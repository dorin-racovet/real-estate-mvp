# Story 1.7: Frontend Authentication & Admin Dashboard

**Epic:** 1. Foundation & Authentication
**Status:** In Progress

## Story Statement
As an admin, I want to log in to a web dashboard so that I can view and manage agents.

## Acceptance Criteria
1. **Login Page**:
    - Form with Email and Password.
    - Submits to `POST /api/v1/auth/access-token` (form-data).
    - Handles errors (invalid credentials).
    - Stores JWT token in localStorage/Context.
    - Redirects to Dashboard on success.
2. **Layout & Navigation**:
    - Sidebar/Header with "Logout" button.
    - Protected Routes (redirect to /login if unauthenticated).
3. **Admin Dashboard (Home)**:
    - fetch `GET /api/v1/admin/agents`.
    - Display list of agents (Name, Email, Phone, Created At).
4. **Create Agent**:
    - Button "Create Agent".
    - Modal or Page with form (Email, Password, Name, Phone).
    - `POST /api/v1/admin/agents`.
    - Refreshes list on success.
5. **Tech Stack**: React, TypeScript, Tailwind CSS, React Router, Axios (or fetch).

## Dev Notes
- **API Integration**:
    - Base URL: `http://localhost:8000`.
    - Configure Vite proxy to avoid CORS issues in dev, or use absolute URLs.
- **State Management**:
    - Use React Context for Auth State (`AuthProvider`).
- **Components**:
    - `LoginPage.tsx`
    - `DashboardLayout.tsx`
    - `AgentList.tsx`
    - `CreateAgentForm.tsx`
- **Tailwind**:
    - Use standard Tailwind utility classes for basic styling.
